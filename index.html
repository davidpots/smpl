This is SMPL

a non-intrusive css framework that respects your design

download on GitHub

created by jared cunha

Powered by SASS

All grid properties use SASS mixins, so you can use the framework classes or apply your own.

Fits Your Grid

Most CSS frameworks are built with a pre-defined grid. Your site is unique and SMPL won't get in the way.

Fully Customizable

SMPL includes a few common layout divisions. It's your site, so you set your own breakpoints.

Demonstration

SMPL uses class names that describe behaviors rather than column counts on a predefined grid.

Getting Started

What it is: Intentionally limited. Common divisions included, but you should add what you need. What it isn’t: This framework was not built to establish an entire site. It is a small set of class and SASS mixins to get you on your way.

Step 1. Apply your grid columns and gutters

  1. /* Gutters */
  2. $gutterWidth: 1.454545454%;
  4. /* Equal Columns */
  5. $halves: 49.272727272%;
  6. $thirds: 32.363636363%;
  7. $quarters: 23.909090909%;
  8. $sixths: 15.454545454%;

Calculate some percentages for gutters and equal column divisions at the widest width of your grid and plug them into the provided SASS variables. You can even add your own. 

  1. /* Set some divisions */
  2. @mixin twoCols {
  3.    width: $halves;
  4.    margin-right: $gutterWidth;
  5. }
  6. @mixin fourCols {
  7.    width: $quarters;
  8.    margin-right: $gutterWidth;
  9. }
  11. /* 1 Column -> 2 Columns -> 4 Columns */
  12. @mixin colItem-1to2to4_Break1 {
  13.    @include twoCols;
  14. }
  15. @mixin colItem-1to2to4_Break2 {
  16.    @include fourCols;
  17. }

Those variables will then apply to a series of mixins and includes that will be used in your site's breakpoints. At Break 1 (referring to the content’s first breakpoint), content will display in two columns. Break 2 will then display the content down further into four columns.

Step 2. The scaffolding

  1. <div class="cols-1-2-4">
  2.   <div class="col-item col-item-1"></div>
  3.   <div class="col-item col-item-2"></div>
  4.   <div class="col-item col-item-3"></div>
  5.   <div class="col-item col-item-4"></div>
  6. </div>

Parent class names describe exactly how the columns will expand. .cols-1-2-4 will go from 1 column, to 2 columns, to 4 columns.

Step 3. Set some breakpoints

  1. /* Expansion A =========== */
  2. @media screen and (min-width: 600px) {
  3.    .cols-1-2-4 .col-item {
  4.       @include colItem-1to2to4_Break1;
  5.    }
  6.    .cols-1-2-4 .col-item-2,
  7.    .cols-1-2-4 .col-item-4 {
  8.       @include lastInRow;
  9.    }
  10. }
  12. /* Expansion B =========== */
  13. @media screen and (min-width: 900px) {
  14.    .cols-1-2-4 .col-item {
  15.       @include colItem-1to2to4_Break2;
  16.    }
  17.    .cols-1-2-4 .col-item-4 {
  18.       @include lastInRow;
  19.    }
  20. }

In Expansion A, .cols-1-2-4 reaches Break 1, and content is displayed in two columns as defined in Step 1. In Expansion B, it reaches Break 2, and content is displayed in four columns.. The last item in each row should have the $lastInRow include to remove right margins.

You should set your media queries to whatever your design requires.

  1. /* Expansion A =========== */
  2. @media screen and (min-width: 600px) {
  3.    .photos-list .photo {
  4.       @include colItem-1to2to4_Break1;
  5.    }
  6.    .photos-list .photo-2,
  7.    .photos-list .photo-4 {
  8.       @include lastInRow;
  9.    }
  10. }
  12. /* Expansion B =========== */
  13. @media screen and (min-width: 900px) {
  14.    .photos-list .photo {
  15.       @include colItem-1to2to4_Break2;
  16.    }
  17.    .photos-list .photo-4 {
  18.       @include lastInRow;
  19.    }
  20. }

Also, if you’re not a fan of framework class names, you can always use the SASS includes with your own.
